# Decision Rationale Template

Use this block in service docs when introducing or changing architecture choices.

## Decision
- State the choice in one sentence.

## Context
- Describe constraints, failure modes, and requirements.

## Alternatives Considered
- Option A: why not chosen.
- Option B: why not chosen.

## Tradeoffs
- Benefits gained.
- Costs accepted.

## Consequences
- Immediate impact on code, tests, and operations.
- Long-term implications and possible extraction/migration concerns.

## Evidence
- Link to relevant code paths and spec sections.
